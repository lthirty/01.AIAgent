# RK3588 产品详细设计文档（详细版）

文档编号：RK3588-HW-DDD-001  
文档版本：v1.0.0  
发布日期：2026-03-03  
适用阶段：方案评审 / 原理图评审 / PCB评审 / EVT / DVT  
适用对象：硬件设计、PCB设计、SI/PI、测试、DFM、项目管理

## 1. 设计目标与边界

### 1.1 总体目标
- 构建基于 RK3588/RK3588S 的可量产硬件平台。
- 保障关键能力：稳定启动、DDR稳定、高速接口稳定、热稳定、EMC可收敛。
- 形成可复用设计资产：模块规则、约束表、检查表、验证计划。

### 1.2 关键约束
- 成本约束：在性能目标满足前提下控制 BOM 成本和工艺复杂度。
- 进度约束：EVT、DVT、PVT 按里程碑推进，P0风险不得带入下阶段。
- 质量约束：关键链路必须可测可追溯，问题必须闭环到 Owner 和截止日期。

### 1.3 本文档不覆盖项
- 结构件详图和整机模具细节。
- 全部法规认证终版报告（仅给预验证策略）。

## 2. 系统架构定义

### 2.1 模块分层
- 计算核心层：RK3588 SoC、DDR、启动存储（eMMC/SPI/NVMe）。
- 连接扩展层：USB3.x、PCIe、RGMII/SGMII、MIPI、HDMI/DP（按项目裁剪）。
- 支撑保障层：电源树、时钟树、复位/启动链、调试口、测试点。

### 2.2 关键数据流和控制流
- 启动流：Boot strap -> ROM -> 启动介质 -> DDR训练 -> 内核加载。
- 高速数据流：DDR/PCIe/USB/MIPI/显示链路。
- 监控流：PG/FAULT/温度/复位信号回传，用于故障定位。

## 3. 电源详细设计

### 3.1 电源轨规划原则
- SoC 核心域、IO域、DDR域分轨供电，避免互扰。
- 每条关键 rail 定义：电压、容差、峰值电流、纹波、瞬态、上电顺序。
- 电源设计必须支持“波形可测、问题可复现、整改可验证”。

### 3.2 电源轨参数表（设计输入）

| Rail | 目标电压 | 容差 | 峰值电流 | 纹波目标 | 上电顺序 | 备注 |

| VDD_CPU | 待定 | 待定 | 待定 | 待定 | 按HWDG | |
| VDD_GPU | 待定 | 待定 | 待定 | 待定 | 按HWDG | |
| VDD_NPU | 待定 | 待定 | 待定 | 待定 | 按HWDG | |

| 1V8/3V3_AO | 待定 | 待定 | 待定 | 待定 | 先上电 | |

### 3.3 SCH 检查要点
- 电源树、使能链、PG/FAULT 拓扑完整且命名一致。
- PMIC 补偿网络与反馈分压有依据可追溯。
- SoC/DDR/NPU 电源脚去耦网络（高频+中频+储能）完整。
- 关键 rail 预留测试点和故障注入点。

### 3.4 PCB 检查要点
- PMIC 开关环路最小化，电感/输入输出电容就近闭环。
- 大电流路径短、宽、连续，过孔阵列满足电流密度。
- 关键回流路径不跨分割，避免环路增大导致 EMI。

### 3.5 验证要求
- 上电时序抓波：全轨并发采样并对照时序窗。
- 纹波测试：空载/典型/满载+高低温。
- 瞬态测试：动态负载阶跃验证恢复时间与过冲。

## 4. 时钟与复位启动详细设计

### 4.1 时钟
- 主晶振、RTC、接口参考时钟分别定义精度和抖动要求。
- 晶振布局原则：短、对称、连续参考面、远离噪声源。
- 时钟线禁止跨缝回流，必要时设置保护地策略。

### 4.2 复位与启动
- Boot strap 必须与启动介质配置一致。
- POR/BOR/WDT 必须形成可恢复链路，禁止不可控死锁状态。
- 必须定义异常恢复流程（MaskROM/下载模式）。

### 4.3 验证要求
- 冷启动、热启动、反复启停、断电恢复测试。
- 毛刺注入和边界供电测试，评估启动鲁棒性。

## 5. DDR 子系统详细设计

### 5.1 设计输入
- DDR 类型、容量、位宽、速率、拓扑、终端策略。
- SI/PI 约束：阻抗、等长、串扰、回流完整性。

### 5.2 SCH 检查要点
- VREF/VTT 独立供电与去耦完整。
- DQ/DQS/CLK/CA 分组约束定义完整。
- ODT/RTT/终端网络设置与手册一致。

### 5.3 PCB 检查要点
- Byte lane 分区清晰，走线按组约束。
- 减少过孔和stub，控制层切换次数。
- 关键网络参考层连续，避免跨分割。

### 5.4 验证要求
- DDR 训练日志检查（常温+高低温）。
- 长稳与压力测试，统计错误率与重训练情况。

## 6. 高速接口详细设计

### 6.1 接口范围
- USB3.x、PCIe、MIPI、HDMI/DP、RGMII/SGMII（按项目裁剪）。

### 6.2 通用规则
- 阻抗连续、差分对内匹配、回流路径连续。
- 限制过孔、避免90度拐角、减少不连续结构。
- 接口防护器件靠近连接器，兼顾带宽与防护能力。

### 6.3 关键约束表（示例）

| 接口 | 阻抗目标 | 等长策略 | 最大过孔 | ESD策略 | 备注 |

| USB3.x | 差分90R | 对内匹配 | 待定 | 接口侧TVS | |
| PCIe | 差分85R | Lane内匹配 | 待定 | 接口侧防护 | |
| MIPI | 差分100R | Lane内匹配 | 待定 | 视接口类型 | |
| RGMII | 单端50R | 时钟与数据匹配 | 待定 | 网口侧防护 | |

### 6.4 验证要求
- 链路训练、误码、吞吐、插拔稳定性。
- 边界温度和供电扰动下稳定性回归。

## 7. PCB 叠层、EMC、热、DFM/DFT

### 7.1 叠层和布局
- 高速层邻接完整参考平面。
- 电源与地层优先保证连续性与低阻抗回流。
- SoC/PMIC/DDR/高速连接器按“信号+热+可制造”协同布局。

### 7.2 EMC/ESD
- 高频开关环路面积最小化。
- 外部接口口的 ESD 路径短、直、连续。
- 预留预扫描整改位置（电阻位、磁珠位、共模位）。

### 7.3 热设计
- 热源器件定义热路径：铜皮、过孔、导热垫、散热器。
- 满载高温场景下不得出现功能性降级故障。

### 7.4 DFM/DFT
- 工艺窗口：线宽线距、孔径、阻抗公差对齐板厂能力。
- ICT/FCT：关键电源、时钟、接口必须具备可达测试点。
- 测试脚本与治具版本纳入配置管理。

## 8. 验证与里程碑

### 8.1 EVT 门禁
- 可稳定启动。
- DDR与关键接口基本稳定。
- P0 问题可定位且有整改计划。

### 8.2 DVT 门禁
- 关键功能和稳定性测试通过。
- 热和EMC预验证无阻断问题。
- P0=0，P1 有关闭计划与截止日期。

### 8.3 PVT 门禁
- 可制造性和一致性满足量产要求。
- 产线测试方案和判定标准冻结。
